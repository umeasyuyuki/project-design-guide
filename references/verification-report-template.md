# 検証レポートテンプレート

本テンプレートは `project-design-guide` スキルの実装モード（Mode C）で
検証レポートを生成する際の構成ガイドラインである。

---

## ドキュメント構成

### 1. 検証概要

```markdown
## 1. 検証概要

| 項目 | 内容 |
|:---|:---|
| プロジェクト名 | `<project-name>` |
| 検証日 | YYYY-MM-DD |
| 検証トリガー | Phase C3（全機能実装完了後） / Phase C4（機能追加後） |
| 総合判定 | READY / NOT READY |
```

### 2. 検証結果サマリー

```markdown
## 2. 検証結果サマリー

| # | 検証項目 | 結果 | 詳細 |
|:--|:---------|:-----|:-----|
| 1 | Build | PASS / FAIL | エラー数、警告数 |
| 2 | Type Check | PASS / FAIL | エラー数 |
| 3 | Lint | PASS / FAIL | エラー数、警告数 |
| 4 | Test Suite | PASS / FAIL | テスト数、通過数、カバレッジ% |
| 5 | Security Scan | PASS / FAIL | 検出された問題数 |
| 6 | 設計整合性 | PASS / FAIL | 実装済み機能数 / 全機能数 |
```

### 4. 総合判定

```markdown
## 4. 総合判定

**判定: READY / NOT READY**

### READY の場合
- 全項目 PASS
- 残課題（Warning レベル）があれば一覧化

### NOT READY の場合
- 修正が必要な項目の一覧
- 各項目の推奨修正方法
- 修正後の再検証手順
```

## 検証実行のガイドライン

### 実行タイミング
- **Phase C3**: 全機能実装完了後の包括的検証
- **Phase C4 Step 5**: 機能追加後のリグレッション込み検証
- **任意のマイルストーン**: ユーザーが途中検証を要望した場合

### 判定基準
- **READY**: 全6項目が PASS（Lint の Warning のみ許容）
- **NOT READY**: 1項目でも FAIL がある場合

### リグレッション確認（機能追加時）
機能追加後の検証では、以下を特に注意する:
- 既存テストが全て引き続き PASS すること
- カバレッジが下がっていないこと（新コードにもテストが書かれていること）
- 既存APIのレスポンス形式が変わっていないこと
