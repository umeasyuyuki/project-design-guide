# コードレビューレポート テンプレート

本テンプレートは `project-design-guide` スキルの分析モード（Mode B）で
コードレビューレポートを生成する際の構成ガイドラインである。

---

## ドキュメント構成

### 1. プロジェクト概要（自動検出結果）

```markdown

| 項目 | 内容 |
|:---|:---|

| 言語 | Python 3.11 |
| フレームワーク | FastAPI 0.104 |
| データベース | PostgreSQL 15 (SQLAlchemy) |
| インフラ | GCP (Cloud Run, Cloud SQL) |
```

### 2. レビューサマリー

発見事項の全体像を表形式で示す:

```markdown
## 2. レビューサマリー

| 重要度 | 件数 | 主な分野 |
|:---|:---|:---|
| Critical | 2件 | セキュリティ、データ整合性 |
| Warning | 5件 | アーキテクチャ、エラーハンドリング、DB設計 |
| Info | 3件 | コード品質、モダン化提案 |
```

```markdown

- **対象ファイル**: `src/config.py:23`
- **現状**:
  （問題のあるコードを引用）
- **リスク**:
  （何が起こりうるかを具体的に）
- **改善案**:
  （修正後のコード例）
- **対応方針**: 修正する / TODO / 対応不要（理由）

> **なぜこれが問題か？**
> （教育コメント）

> **もっと学ぶなら:**
> - （関連トピック）
```

ユーザーとの合意結果を記録:


## レビュー観点チェックリスト

分析時に必ず確認する項目:

### アーキテクチャ
- [ ] 責務の分離（Controller / Service / Repository）
- [ ] レイヤー構造の明確さ
- [ ] 循環依存の有無
- [ ] スケーラビリティの阻害要因

### セキュリティ
- [ ] 認証情報のハードコード
- [ ] インジェクション脆弱性（SQL, XSS, Command）
- [ ] 入力バリデーション
- [ ] 認証・認可の仕組み
- [ ] CORS設定

### データベース
- [ ] 正規化レベル
- [ ] インデックス設計
- [ ] マイグレーション管理
- [ ] 型選定の妥当性（特に金額、日時）
- [ ] 制約（FK, NOT NULL, UNIQUE）

### エラーハンドリング
- [ ] 空の catch ブロック
- [ ] ユーザー向けエラーメッセージ
- [ ] リトライ戦略
- [ ] ロギングの十分さ

### コード品質
- [ ] 命名規則の統一
- [ ] DRY原則の遵守
- [ ] テストの有無とカバレッジ
- [ ] 未使用コード/依存パッケージ
- [ ] コメント/ドキュメントの質

### インフラ・運用
- [ ] 環境変数管理
- [ ] CI/CD
- [ ] ログ出力方針
- [ ] デプロイの再現性
- [ ] 監視/アラート
