# project-design-guide

Claude Code 用のスキル。ふわっとした要件から設計書を作成し、TDD で実装まで一貫サポートする。

## 概要

プロのエンジニアの視点で **要件定義書・仕様書・DB設計書** をユーザーとの対話を通じて段階的に作成し、さらにその設計書を元に **TDD ベースの実装** まで遂行する。

各設計判断には「なぜこの選択をしたか」の理由と、「もっと学ぶなら」のアドバイスを併記するため、駆け出しエンジニアの学習にも役立つ。

## 3 つのモード

| モード | 用途 | 出力物 |
|:-------|:-----|:-------|
| **A) 新規モード** | ふわっとした要件から設計書をゼロから作る | 要件定義書・仕様書・DB設計書 |
| **B) 分析モード** | 既存プロジェクトのコードを読んで設計書を作る（改善提案付き） | 要件定義書・仕様書・DB設計書・コードレビューレポート |
| **C) 実装モード** | 作成済みの設計書を元に TDD で開発する | 実装計画書・検証レポート・実装コード |

## インストール

```bash
# スキルディレクトリにクローン
git clone https://github.com/umeasyuyuki/project-design-guide.git ~/.claude/skills/project-design-guide
```

## 使い方

Claude Code で以下のように呼び出す:

```
/project-design-guide
```

または、以下のような自然言語でも起動する:

- 「こういうシステムを作りたいんだけど」 → 新規モード
- 「このプロジェクトのドキュメントを作りたい」 → 分析モード
- 「設計書を元に実装して」 → 実装モード

## ワークフロー

### 新規モード (Mode A)

```
Phase 0: プロジェクト情報の確認
Phase 1: ヒアリング（5W1H で要件を深掘り）
Phase 2: 要件定義書の作成
Phase 3: 技術選定とアーキテクチャ設計
Phase 4: 仕様書の作成
Phase 5: DB設計書の作成
  → 完了後、実装モードへの遷移を提案
```

### 分析モード (Mode B)

```
Phase B0: プロジェクトスキャン（技術スタック自動検出）
Phase B1: コードレビュー＆疑問点の洗い出し
Phase B2: 要件定義書の作成（逆引き＋改善提案付き）
Phase B3: 技術選定の評価と改善提案
Phase B4: 仕様書の作成（現状＋改善後の姿）
Phase B5: DB設計書の作成（現状分析＋改善提案）
  → 完了後、実装モードへの遷移を提案
```

### 実装モード (Mode C)

```
Phase C0: 設計ドキュメント読み込み & 実装計画
Phase C1: プロジェクトセットアップ
Phase C2: 機能実装（イテレーティブ TDD サイクル）
Phase C3: 統合検証 & 品質ゲート（6段階）
Phase C4: 機能追加ワークフロー（設計書更新 → TDD 実装）
```

## 出力ファイル

すべて `<repo-root>/docs/` に出力される:

| ファイル | モード |
|:---------|:-------|
| `<slug>_requirements_v1.md` | A / B |
| `<slug>_specification_v1.md` | A / B |
| `<slug>_database_design_v1.md` | A / B |
| `<slug>_code_review_v1.md` | B のみ |
| `<slug>_implementation_plan_v1.md` | C |
| `<slug>_verification_report_v1.md` | C |
